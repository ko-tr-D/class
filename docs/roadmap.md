# Roadmap

## Phase 0. 구조 확정

- 웹앱 방향 확정
- 데이터 모델 초안 작성
- 보안 모델 작성
- 교사용/학생용 화면 흐름 정의

## Phase 1. 교사용 MVP

- 교사용 이메일 로그인
- 학년도/반/학생 관리
- PDF 업로드
- OCR 결과 검수 화면
- 수업 자료를 단원/성취기준과 연결

상태: 브라우저 SPA 프로토타입에서 기본 흐름 구현.

## Phase 2. 평가와 기록

- 평가 기준표 관리
- 학생별 근거 기록
- 성취 수준 입력
- 학생 기록 카드
- 생활기록부 문구 초안 생성
- 교사 수정/승인 이력 저장

상태: 평가 기준표, 학생별 평가 입력, 문구 승인 흐름 구현. AI 초안 생성은 이후 API 연동 단계에서 구현.

## Phase 3. 학생용 문제 풀이

- 학생 접속 코드
- 학생용 평가 화면
- 객관식/단답형/서술형 문항
- 제출과 채점
- 풀이 이벤트 로그

상태: 학생 접속 코드, 객관식/단답형 풀이, 제출 기록, 답안 수정 횟수 저장 흐름 구현.

## Phase 4. 분석

- 학생별 성취도 흐름
- 반별 오답 집중 문항
- 단원/성취기준별 약점
- 풀이 시간과 수정 패턴
- 분석 알고리즘 버전 관리

상태: 정답률, 풀이 시간, 답안 수정 횟수 기반 분석실 구현. 알고리즘 버전은 v0.1 규칙 기반으로 표시.

## Phase 5. 운영

- 백업/복구
- 학년도 이관
- 데이터 내보내기
- 학교 계정 연동 검토
